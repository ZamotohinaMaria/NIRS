# my-GINE-model

Train a GINE classifier on API-call sequence datasets.

## 1) Environment

Use Python 3.10+.

Install base dependencies:

```bash
pip install numpy pandas scikit-learn torch
pip install torch-geometric
```

If `torch-geometric` installation fails for your CUDA/PyTorch build, install it using official wheels from PyG docs.

## 2) Quick start (MalBehavD-V1)

From repo root:

```bash
python .\my-GINE-model\train_gine.py --csv .\MalbehavD-V1-main\MalBehavD-V1-dataset.csv --dataset-format auto --include-unnamed-api-cols --epochs 30 --batch-size 64
```

This dataset is in wide format (`0..N` API columns), so `auto` should resolve to `wide_api`.
By default, outputs are saved under `runs/`.

Train on all provided rows without internal split:

```bash
python .\my-GINE-model\train_gine.py --csv .\runs\splits\train_1300.csv --dataset-format auto --include-unnamed-api-cols --no-internal-split
```

## 3) Run with one text sequence column

If your dataset has one sequence column like `api_sequence`:

```bash
python .\my-GINE-model\train_gine.py --csv .\data\your_dataset.csv --dataset-format single_seq --label-col label --seq-col api_sequence
```

## 4) Dataset formats

- `--dataset-format auto`
  - Auto-detect between `single_seq` and `wide_api`.
- `--dataset-format single_seq`
  - Use one text column (`--seq-col`) where tokens are split by `--delimiter-regex`.
- `--dataset-format wide_api`
  - Use many API columns (default regex `^\d+$`, for columns `0,1,2,...`).
  - Optional: `--include-unnamed-api-cols` to include non-empty trailing `Unnamed:*` columns.

## 5) Main outputs

- Output base folder: `runs/` (override via `--output-dir`)
- Best model weights: `runs/models/<auto-generated-name>.pt`
  - Auto name is compact and based on key training parameters.
  - After training it also includes best epoch and val F1.
  - You can still override with `--save-path`.
- API vocabulary: `runs/vocab/api_vocab.json` (override via `--vocab-path`)
- Run summary text: `runs/results/<auto-generated-name>.txt` (override via `--results-path`)
- Console metrics:
  - validation: accuracy, F1, ROC-AUC per epoch
  - test: accuracy, precision, recall, F1, ROC-AUC

Relative `--save-path`, `--vocab-path`, and `--results-path` are resolved inside `--output-dir`.


## 7) Inference (classification)

Predict one sequence:

```bash
python .\my-GINE-model\predict_gine.py --model-path .\runs\models\g_h128l3_b64e30_lr0p001_s42_xxxxxxxx_be15_vf922.pt --vocab-path .\runs\vocab\api_vocab.json --input-seq "NtOpenFile NtReadFile NtClose"
```

Predict from CSV and compute metrics:

```bash
python .\my-GINE-model\predict_gine.py --model-path .\runs\models\g_h128l3_b64e30_lr0p001_s42_xxxxxxxx_be15_vf922.pt --vocab-path .\runs\vocab\api_vocab.json --input-csv .\MalbehavD-V1-main\MalBehavD-V1-dataset.csv --dataset-format auto --include-unnamed-api-cols --label-col labels
```

Batch output is a metrics report saved to `runs/predictions/<auto-generated-name>.txt` unless `--predictions-path` is provided.

## 8) Multi-split benchmark

Run several random splits and get averaged metrics:

```bash
python .\my-GINE-model\run_splits.py --csv .\MalbehavD-V1-main\MalBehavD-V1-dataset.csv --label-col labels --dataset-format auto --include-unnamed-api-cols --train-per-class 650 --num-runs 5 --base-seed 42
```

The script always creates a new experiment folder:

`runs/multi_split_eval/exp_<timestamp>/`

It stores:
- per-run train/eval CSV splits
- per-run model checkpoints
- per-run vocab files
- per-run metrics reports
- `summary/summary.csv`
- `summary/aggregate.txt` (mean/std/min/max metrics)
